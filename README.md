# agent-memory-runtime
适用于长时间运行的 AI 代理的生产就绪内存运行时环境


##为什么要创建这个项目

LLM agent普遍会面临这些问题:

1. 上下文窗口过段：模型单次能处理的 Token 数量有限，导致长文本或复杂任务无法一次性输入。
2. 跨会话记忆丢失：多次独立对话或长时间断开后，模型会遗忘历史状态和用户偏好
3. 检索噪声(记忆污染/注意力分散): 长期记忆或外部知识库在提取时，混入了错误的、过时的或无关的记忆，污染了当前的思考/上下文太长时agent的注意力跑偏，抓不住重点
4. Token消耗量大：多轮对话，复杂的prompt，ReAct循环，注意力涣散导致的Token消耗量激增


#优化

1.短期记忆 + 长期记忆
2.语义记忆检索
3.记忆排序
4.内存整合
5.上下文优化
